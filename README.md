# E-commerce Testing Project

## Project Overview

This project demonstrates manual testing of a sample e-commerce web application. The focus was on validating core functionalities such as login, product selection, cart operations, and checkout process.

## Modules Covered

* Login (valid/invalid credentials, validation checks)
* Product (search and selection)
* Cart (add/remove items, quantity update)
* Checkout (address, payment method, order confirmation)

## Test Artifacts

* Test Cases (positive & negative scenarios)
* Bug Report (with steps, expected vs actual result)
* Test Summary Report
* Test Data (valid and invalid inputs)

## Tools Used

* Excel (test cases & test data)
* Browser (manual testing)

## Test Approach

* Designed test cases based on requirements
* Executed test cases and recorded results (Pass/Fail)
* Identified defects and documented them clearly
* Performed basic regression testing after fixes

## Key Learning

* Understanding of SDLC and STLC
* Writing and executing test cases
* Basic defect reporting process
* Importance of validation and edge cases
